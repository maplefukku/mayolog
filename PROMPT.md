# MayoLog デザイン仕上げタスク

## 背景
MayoLog（マヨログ）は「迷った瞬間を5秒で記録するメモアプリ」です。
framer-motionアニメーション、ダークモード、レスポンシブ対応は既に実装済みです。
デザイン仕上げとして、DESIGN_SYSTEM.mdの禁止事項に違反している箇所を修正してください。

## 修正が必要な箇所

### 1. アイコンだけのボタン（ラベルなし）を修正
**場所:** `src/app/input/page.tsx` の履歴ボタンと判断軸ボタン

**現状:**
```tsx
<Button variant="ghost" size="icon" aria-label="履歴">
  <History className="size-4" />
</Button>
<Button variant="ghost" size="icon" aria-label="判断軸">
  <BarChart3 className="size-4" />
</Button>
```

**問題:** DESIGN_SYSTEM.md禁止事項8「アイコンだけのボタン（ラベルなし）— 分かりにくい」

**修正案:**
- 履歴ボタンと判断軸ボタンを削除する（MVPでは不要な機能）
- または、ラベル付きボタンに変更する

### 2. 日本語UI品質チェック
- 全てのテキストが自然な日本語か確認
- 翻訳くさくないか確認
- エラーメッセージが具体的か確認

### 3. その他の禁止事項チェック
以下を確認し、違反があれば修正:
1. グラデーション背景 — なし
2. 影が濃すぎる — shadow-lg以上は原則禁止
3. ボーダーが目立ちすぎる — border-border/50で薄く
4. 色を3色以上使う — グレースケール + アクセント1色
5. 角丸がバラバラ — rounded-2xlかrounded-fullに統一
6. フォントサイズの乱立 — スケールに準拠
7. 余白が狭い — p-4未満のパディングは使わない
8. アイコンだけのボタン（ラベルなし）— 分かりにくい
9. 英語のまま残す — エラーメッセージ含め全て日本語
10. デフォルトのshadcn/uiそのまま — 必ずカスタマイズする

## 作業プロセス

1. 上記の修正が必要な箇所を修正する
2. npm run build でビルドが通ることを確認
3. 修正内容を要約して報告する

## 注意事項
- 大きな変更はしない（デザイン仕上げのみ）
- 既存のアニメーション、ダークモード、レスポンシブ対応は維持する
- テストがあれば実行して確認する
