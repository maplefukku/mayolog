# MayoLog（マヨログ）UI実装

## プロジェクト概要
迷った瞬間を5秒で記録するメモアプリ。「迷い」を入力するだけで、AIが判断パターンを分析し、自分が何を重視して選んできたのかという"軸"が可視化される。

## 技術スタック
- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- shadcn/ui + framer-motion
- next-themes (ダークモード)
- GLM API (glm-4.7) — OpenAI互換

## デザインルール（DESIGN_SYSTEM.md準拠）

### カラー
- ライト: --background: #FFFFFF, --foreground: #0A0A0A
- ダーク: --background: #0A0A0A, --foreground: #FAFAFA
- アクセント1色のみ（黒または白）

### タイポグラフィ
- ヒーロー見出し: `text-4xl font-bold tracking-tight`
- セクション見出し: `text-2xl font-semibold`
- 本文: `text-base leading-relaxed`
- 補足: `text-sm text-muted-foreground`

### コンポーネント
- ボタン: `rounded-full h-12`
- カード: `rounded-2xl border bg-card p-6 shadow-sm hover:shadow-md`
- 入力欄: `rounded-2xl border-border/50 bg-muted/50 min-h-[120px]`
- max-width: `max-w-lg` (512px)

### アニメーション（framer-motion）
```tsx
// ページ遷移
initial={{ opacity: 0, y: 20 }}
animate={{ opacity: 1, y: 0 }}
transition={{ duration: 0.4, ease: [0.25, 0.46, 0.45, 0.94] }}
```

## 実装タスク

### Phase 1: 基盤構築（15分）
1. globals.cssにテーマ変数を設定（ダークモード対応）
2. ThemeProviderを設定（next-themes）
3. Header コンポーネント作成
4. レイアウト構築

### Phase 2: 各画面UI実装（25分）
1. ランディングページ（LP）
2. 迷い入力画面
3. AI質問画面（モック）
4. 分析結果画面（モック）

### Phase 3: アニメーション・仕上げ（5分）
1. framer-motionでページ遷移アニメーション
2. プレースホルダーの5秒切替
3. ダークモード動作確認
